### ✨ 包含的 Skills

#### 📝 教程写作助手 (Tutorial Writing)

**描述**：针对某一领域或话题进行结构化写作（教程、博客、讲解文等）。当用户希望写技术/领域文章、知识点讲解或任何需要「先想清楚再写」的成文时使用。引导作者明确写作目标与受众，基于一手资料写作，输出简洁清晰的 Markdown。

**适用场景**：

* 需要写技术教程或入门文章
* 想写某知识点的讲解、对比分析（如「A 和 B 有啥区别」）
* 写博客前思路不清晰，需要先理清目标和受众
* 希望成文深入浅出、有示例、可验证（代码可运行）

**核心功能**：

* 📋 理解领域并引导作者：澄清话题、目标读者、重点与篇幅偏好
* 🔍 搜集资料：优先官方文档与权威来源，避免不可溯源内容
* ✍️ 正式写作：深入浅出、先示例再概念、一段一意、Markdown 简洁
* 🎯 守则约束：不啰嗦不重复、术语浅显一句再展开、代码块可运行

**触发方式**：

```
帮我写一篇 XX 教程
写个关于 XX 的博客/讲解
我想写技术文章，先帮我理清思路
/tutorial-writing
```

### 🚀 快速开始

#### 安装方式

**手动安装**

```bash
# 克隆本仓库（可放到任意目录，或 ~/.cursor/skills/ 等）
git clone https://github.com/wingchi-leung/how-to-skills-hub.git
```

也可只下载或复制 [tutorial-writing-zh/SKILL.md](./tutorial-writing-zh/SKILL.md) / [tutorial-writing-en/SKILL.md](./tutorial-writing-en/SKILL.md) 到你的 Skills 目录。

#### 使用 Skills

在对话中直接说明写作需求，或引用技能文件名，例如：

```
用 tutorial-writing 技能，帮我写一篇 Python 异步入门
写一篇关于 XX 的教程，受众是……
```

或把对应 `.md` 的全文作为上下文粘贴进对话，再说明话题与受众。
