#### üìù Tutorial Writing Assistant

**Description**: Structured writing for a given domain or topic (tutorials, blogs, explainers). Use when you want to write a tech/domain article, explain a concept, or any piece that benefits from "think first, then write." Guides you to clarify goals and audience, use primary sources, and output clear, concise Markdown.

**Use Cases**:

* Need to write a technical tutorial or intro article
* Want to explain a concept or compare two things (e.g. "what's the difference between A and B")
* Ideas are unclear before writing; need to clarify goal and audience first
* Want output that is easy-to-grasp then deep, with runnable code examples

**Core Features**:

* üìã Understand the domain and guide the author: clarify topic, target reader, focus, and length preferences
* üîç Gather sources: prefer official docs and authoritative sources; avoid unverifiable content
* ‚úçÔ∏è Write the tutorial: easy then deep, example before concept, one idea per paragraph, minimal Markdown
* üéØ Guidelines: no repetition, one clear sentence for terms then expand if needed, runnable code blocks

**Trigger**:

```
Help me write a tutorial on XX
Write a blog post / explainer about XX
I want to write a tech article, help me clarify my thoughts first
/tutorial-writing
```

Or paste the full content of the corresponding `.md` file into context, then state your topic and audience.
